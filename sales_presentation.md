## スライド2：なぜ今、AI開発なのか？

### プログラミングの常識が変わった

**2023年まで：プログラミング = 専門家の仕事**
```
✗ コードを1行ずつ書く
✗ エラーが出たら自分で調べて修正
✗ 習得に数年かかる
✗ 非エンジニアには無理

結果：システム開発はエンジニアに依頼するしかない
```

**2024年以降：AI × 誰でも = 開発者**
```
✓ 日本語で指示するだけでコードが生成される
✓ エラーが出てもAIが原因を教えてくれる
✓ 3週間で実用的なツールが作れる
✓ 非エンジニアでも開発可能

結果：現場の人が自分でツールを作れる時代に
```

### Cursor = AIペアプログラミング

**Cursorとは？**
```
AI搭載の開発環境（IDE）

まるで優秀なエンジニアが隣にいるような感覚：
・「こういう機能が欲しい」→ AIがコードを生成
・「エラーが出た」→ AIが原因を分析して修正案を提示
・「このコードどういう意味？」→ AIが解説してくれる
```

**実際の開発の流れ**
```
従来：
1. プログラミング言語を学習（数ヶ月〜数年）
2. ドキュメントを読んで理解
3. コードを1行ずつ書く
4. エラーが出たら試行錯誤
5. 何日もかかる

Cursorを使った開発：
1. Cursorに「〇〇を作って」と日本語で指示
2. AIがコード全体を生成（数秒〜数分）
3. エラーが出たらAIに聞く
4. AIが修正案を提示
5. 数時間で完成

約100倍速い！
```

```
AIがやること：
・コードを書く

あなたがやること：

## スライド3：Cursorの使い方イメージ

### Cursor画面の基本構成

```
┌─────────────────────────────────────────┐
│ Cursor                            [─][□][×]│
├─────────────────────────────────────────┤
│ Explorer  │  エディタ画面  │  Chat/Composer│
│           │                │               │
│ 📁project │  コードが      │  AIに質問や   │
│  📄main.py│  表示される    │  指示を出す   │
│  📄app.py │  エリア        │  エリア       │
│  📁data   │                │               │
│           │                │               │
└─────────────────────────────────────────┘
```

### 3つの主要機能

**1. Chat機能（Cmd+L）**
```
使い方：
コードを選択して「このコードを説明して」
エラーメッセージをコピーして「このエラーを修正して」

AI が即座に回答：
・コードの説明
・エラーの原因分析
・修正方法の提案
```

**2. Composer機能（Cmd+I）- 最重要！**
```
使い方：
「営業日報をExcelに出力するプログラムを作って」
と入力するだけ

AI が自動で：
・必要なファイルを全て作成
・複数ファイルを同時編集
・実行方法も教えてくれる

数分で完成！
```

**3. Agent機能**
```
AI が自律的に：
・ファイル検索
・コード編集
・実行とテスト
まで自動でやってくれる
```

### 実際の開発フロー

**Step 1：Composerで初期構築**
```
あなた：
「営業日報から顧客情報を抽出してExcelに出力する
Pythonプログラムを作ってください」

Cursor：
→ main.py を作成
→ requirements.txt を作成
→ サンプルの日報ファイルも作成
→ 実行方法を説明

所要時間：約2分
```

**Step 2：動作確認**
```
ターミナルで実行：
$ python main.py

結果：
Excel ファイルが出力された！

→ 中身を確認して、期待通りか確認
```

**Step 3：修正・改善**
```
あなた（Chatで）：
「見込み金額も抽出するように追加して」

Cursor：
→ コードを自動修正
→ 変更箇所を説明

所要時間：約30秒
```

**Step 4：エラーが出たら**
```
エラーメッセージ：
ModuleNotFoundError: No module named 'openpyxl'

あなた（Chatで）：
[エラーメッセージを貼り付け]

Cursor：
「openpyxl がインストールされていません。
以下のコマンドを実行してください：
$ pip install openpyxl」

→ 指示通りに実行
→ 解決！
```

### Cursorの便利機能

**コード生成時の指示例**

```
良い指示（具体的）：
「営業日報テキストから、顧客名・担当者名・見込み金額・
次回アクション日を抽出してExcelに出力するPythonスクリプトを作成。
OpenAI APIのgpt-4o-miniを使用してください」

→ AIが正確に理解して生成

悪い指示（あいまい）：
「営業ツールを作って」

→ AIが何を作ればいいか分からない
```

**Chat vs Composer の使い分け**

```
Chat（Cmd+L）を使う場面：
・質問したいとき
・コードの一部だけ修正したい
・エラーの原因を知りたい

Composer（Cmd+I）を使う場面：
・新しいプロジェクトを作る
・複数ファイルを同時に編集
・大規模な変更をしたい
```

### 学習不要！すぐ使える

```
プログラミング言語の文法を覚える必要なし
↓
日本語で指示するだけ
↓
AIが全部やってくれる

## スライド4：カリキュラムで作るツール

### 営業日報AI整理ツール

**現状の課題**
```
営業担当が毎日書く日報：
「今日はABC株式会社の田中部長を訪問。新システムの提案を行い、
予算500万円で来月の役員会で検討とのこと。次回は2週間後に再訪問予定」

問題点：
・自由記述なので情報がバラバラ
・後で集計するのが大変
・見込み案件の優先順位が不明
```

### Week 1で作るもの：ローカル版

**機能**
```
入力：日報テキストファイル（reports/report_20241105.txt）

処理：Cursor + OpenAI API で自動解析
↓
抽出する情報：
- 顧客名：ABC株式会社
- 担当者：田中部長
- 商談要約：新システム提案
- 見込み金額：500万円
- 確度：B（AIが判定）
- 次回アクション：2週間後に再訪問

出力：Excelファイル（営業日報まとめ_20241105.xlsx）
```

**ファイル構成**
```
sales-report-tool/
  ├── main.py              # メインプログラム
  ├── reports/             # 日報テキストを入れる
  │   ├── report_20241105.txt
  │   └── report_20241106.txt
  ├── output/              # Excel出力先
  └── requirements.txt     # 必要なライブラリ
```

**実行方法**
```bash
# ターミナルで実行
$ python main.py

# 結果
✓ report_20241105.txt を処理しました
✓ report_20241106.txt を処理しました
✓ output/営業日報まとめ_20241105.xlsx を作成しました

→ Excelを開いて確認
```

**Excelの中身（イメージ）**
```
| 日付  | 顧客名  | 担当者  | 商談要約    | 金額   | 確度 | 次回      |

|11/5   |ABC社    |田中部長 |システム提案 |500万   | B    |2週間後    |
|11/6   |XYZ社    |佐藤氏   |CRM導入     |300万   | A    |来週面談   |
```

### デプロイの流れ

**Step 1：GitHubにコードをプッシュ**
```bash
# Cursorのターミナルで実行

# 1. Gitリポジトリ初期化
$ git init

# 2. ファイルを追加
$ git add .

# 3. コミット
$ git commit -m "Initial commit"

# 4. GitHubリポジトリに接続
$ git remote add origin https://github.com/[ユーザー名]/sales-report-web.git

# 5. プッシュ
$ git push -u origin main

→ コードがGitHubにアップロードされる
```

**Step 2：Renderでデータベース作成**
```
Render画面で：
1. 「New +」→「PostgreSQL」を選択
2. Name: sales-db
3. Database: sales_reports
4. Region: Oregon（最も近い）

→ 数分で作成完了
→ 接続URLが発行される（自動で使われる）
```

**Step 3：Webサービスを作成**
```

1. 「New +」→「Web Service」を選択
2. GitHubリポジトリを連携
3. 作成したリポジトリを選択
4. 設定：
   - Name: sales-report-web
   - Environment: Python 3
   - Build Command: pip install -r requirements.txt
   - Start Command: gunicorn app:app
5. 「Create Web Service」をクリック

→ 自動でデプロイ開始
```

**Step 4：環境変数を設定**
```
Render画面の「Environment」タブで：

1. OPENAI_API_KEY
   Value: sk-your-api-key-here

2. DATABASE_URL
   Value: [PostgreSQLの接続URL]
   ※自動で設定される

3. 「Save Changes」をクリック

→ アプリが再起動される
```

**Step 5：デプロイ完了！**
